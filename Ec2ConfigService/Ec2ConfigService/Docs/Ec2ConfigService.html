EC2 Configuration Service

 

Ec2Config service can perform various functions to prepare an instance when it first boots up. Each of these functions can be individually enabled or disabled from the settings form before bundling an instance.

The new settings will be applied to the instance that will be launched from the bundled image.

NOTE: Some functionality is not currently visible through the Ec2ConfigSettings GUI Application.  For modifications to such functionality, a direct edit to the XML files will be necessary.

Set Dns Suffixes:

To Enable/Disable:
Edit CONFIG.XML, <SetDnsSuffixList>, type xs:boolean

Settings details file:
Edit DnsSuffixSettings.XML

Adds entries to your DNS suffix search list to facilitate DNS Lookups.

NOTE: This functionality is key to Windows 2008 Activation!  See “Activate Windows” below for details.

The plugin has the ability to make region-aware decisions on the suffix list based on the availability zone you have launched in.  For example, take these settings from the DnsSuffixSettings.XML file:

  <Suffix regionAware="true">ec2-utilities.amazonaws.com</Suffix>

                

  <RegionMapping>  

      <RegionTag>us-east-1</RegionTag>

      <Prepend>us-east-1.</Prepend>

  </RegionMapping>

When you launch an instance, the following may be present in your meta data at http://169.254.169.254/latest/meta-data/placement/availability-zone:

us-east-1c

The <RegionTag> is a Regular Expression.  When the plugin evaluates “us-east-1c”, it matches and knows to prepend “us-east-1.” to “ec2-utilities.amazonaws.com”, and places “us-east-1.ec2-utilities.amazonaws.com” into your DNS Search List.

The settings pre-configured for you are for KMS Server discover in each region.  You may add settings as you see fit.

Activate Windows:

To Enable/Disable:

Edit Config.XML

Settings details file:

Edit ActivationSettings.XML

Overview

Causes Windows 2008 to attempt activation by searching through the DNS Suffix List for appropriate KMS Server entries. 

In Windows 2008 (non R2), the plugin performs this search manually.  When found, the plugin sets your activation server to the first server to respond to the request successfully.

 In Windows 2008 R2, the auto discovery method built into Windows is able to search the suffix list automatically.

Set Password :  

Sets a random password on the instance, encrypts it with the user launch key and outputs the encrypted password on the console.  This feature is disabled by default after setting up the first password on the instance so that any further reboots or bundles  created from the instance does not change the password set by the user. 

Set ComputerName:

This plugin sets the hostname of the instance to a unique name based on the IP address of the instance. Setting hostname requires a reboot of the instance; if this feature is enabled, your instance will reboot once after booting up.  If you wish to set your own  hostname, or prevent your existing host name from being modified, you should disable this setting.

Initialize Drives:

Initializes and formats all uninitialized drives attached to the instance.  When a new instance is launched, all the ephemeral drives (non-boot drives) attached to the instance are uninitialized. This feature initializes and mounts them as d:, e:, etc.

Set RDP Certificate:  

(Unused in Windows 2008, since Windows 2008 is able to generate its own certificates).

Sets a self signed certificate to the Remote desktop server running on the instance. This allows user to securely RDP into the instance.

Set EventLog Output:  

This plugin outputs Eventlog entries on the console during boot.  The Eventlog entries to output are chosen based on the user-defined settings.  By default, the 3 most recent error entries from the System event log are sent to the console.

Set Drive Letters:

 Sets drive letter of mounted volumes based on the user defined settings.

By default, when an EBS volume is attached to an instance, it may show up as any drive letter on the instance.

To mount an EBS volume to a fixed drive letter, user can map the name (disk Label) of a volume to a particular drive letter in the mappings table.  Then, when a disk with that label appears in Disk Manager, the Ec2Config plugin will automatically change the drive letter to the requested mapping.  This will fail if the drive letter is already in use.  It is recommended that you pick drive letters from bottom up (Z:, Y:..) to avoid this.

Wallpaper Information:

This plugin displays system information on the desktop’s background. By default this runs from the startup folder of all users. To disable this feature uncheck the box in the Ec2Config application.

A settings file titled WallpaperSettings.xml can be modified to change the data that displays on the wallpaper. It has the possibility to display any meta-data found at 169.254.169.254/latest/meta-data/. It also has the possibility to display Machine wide environmental variables. To display an environmental variable specify an element as follows:

<WallpaperInformation>

    <name>Display Name</name>

    <source>EnvironmentVariable</source>

    <identifier>VariableName</identifier>

</WallpaperInformation>

HANDLE USER DATA :

This plugin will read in the user data specified for the instance and then check if it contain the tags <script> and </script> or <powershell> and </powershell>. If it finds either pair, it will take the information between those two tags and save it to a batch file for the “script” tags and powershell file for the “powershell” tags located in the Settings folder of this application. It will then execute the file as the local Administrator during the start of an instance. This feature is automatically enabled during Sysprep.

The scripts will only be created and executed on the first launch of an instance after a sysprep. If you want to have the scripts created and executed again set the Ec2HandleUserdata plugin state to Enabled, or  place a check next to the User Data feature in the UI, then reboot or restart the EC2Config service.

For example, <script>echo 97% completed</script><powershell>invoke-expression ‘echo hello’</powershell will create a batch file that contains “echo 97% completed” and a powershell script that contains “invoke-expression ‘echo hello’”.

Dynamic Root Volume :

After the EC2Config Service has started, this plugin will expand Disk 0/Volume 0 to include any unpartitioned space. Once performed, the Root Volume checkbox within the Disk Settings will be unchecked. This feature is automatically enabled during Sysprep.
